# FFB 디버깅 — 상태와 내일 시퀀스

## 오늘 밤 원격 진단 결과 (2026-07-13 새벽, 실측)

`python tools\ffb_doctor.py` 로 휠 저장 설정을 직접 읽음:

| 설정 | 발견값 | 판정 |
|---|---|---|
| **TRUEFORCE** | **0%** | **스모킹 건 — "가볍고 아무 느낌 없음"의 유력 원인** |
| 프로필 | 온보드 1 | G HUB 설정 무시 상태였음 → **데스크톱으로 전환 완료** |
| FFB 강도 | 5.0 Nm | → **6.0 Nm로 상향 완료** (유지 확인됨) |
| 댐핑 | 0% | 정상 (DD 순수 감각) |
| 회전 범위 | 1080° | 게임 캘리브레이션과 일치하는지만 확인 |
| FFB 필터 | 11 (auto) | 정상 |

- TrueForce 0%는 HID++ 직접 쓰기가 **펌웨어에서 조용히 거부됨** (6가지 파라미터 변형 전부) →
  **G HUB 슬라이더로 설정해야 함** (아래 1단계)
- 추정 경위: SW_ID 충돌 시기 + G HUB 강제종료 반복으로 설정 상태 오염

## 내일 실기 시퀀스 (순서대로, 각 30초)

**0단계 — G HUB에서 (게임 켜기 전):**
- RS50 페이지 > **TRUEFORCE 슬라이더 → 40%** (핵심!)
- 강도 슬라이더가 6.0Nm 근처인지 확인 (제가 이미 휠에 6.0을 써놨지만 G HUB 표시와 동기화 확인)
- 프로필 드롭다운이 "데스크톱"인지

**1단계 — 우리 앱 없이 주행** (G HUB만): 느낌 확인
- 묵직+노면 진동 돌아옴 → 원인은 TrueForce 0%였음, 2단계로
- 여전히 가벼움 → 게임 설정 확인: 조작 > 고급 휠 > 포스 피드백 스케일/SAT 100+, 진동 0 아님

**2단계 — `python -m src.main` (LED 없이)** → 이상 없어야 정상

**3단계 — `start.bat` (전체)** → 이상 생기면 config `update_hz = 6`으로 낮춤

**4단계 — 우측 쏠림이 남으면:**
- 정차 후 손 떼기 → 스스로 돌면 **센터 캘리브레이션**: G HUB 휠 설정 또는 베이스 OLED > Calibration
- 윈도우 사운드 좌우 밸런스 50:50 확인 (TrueForce는 오디오 기반 — 밸런스 틀어지면 비대칭 힘)
- 게임 데드존 좌우 대칭 확인

각 단계 결과를 알려주면 다음 수를 바로 제시. 도구: `python tools\ffb_doctor.py` (읽기),
`--fix` (복구), `--guide` (이 시퀀스 출력).
